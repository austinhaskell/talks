Before you start

Updating cabal

> cabal install cabal cabal-install

  • Builds the latest cabal and places it in ~/.cabal/bin
  • Make sure you have ~/.cabal/bin in your shell's PATH

Starting off a project

> cabal init

  • Creates a .cabal file to start your project
  • The .cabal file contains the settings for your project

Create your sandbox

> cabal sandbox init

  • .cabal-sandbox – folder for libraries and build artifacts
  • cabal.sandbox.config – file with target folders for libraries and build artifacts

Finding libraries

Hackage – hackage.haskell.org

Central package archive for Haskell

Adding libraries

Add to build-depends in your .cabal file

Installing libraries

> cabal install

Useful flags

  • --dry-run – See what Cabal plans to do
  • --only-dependencies – Only installs dependencies and not your library or executable
  • --enable-test – Installs dependencies for any automated test suites
  • -jN – Builds N packages in parallel

Building your app

> cabal build

  • Builds your app and places it in dist/

Running your app

> cabal run

  • Runs the executable defined in your .cabal file
  • If multiple executables exists, you can specify the name using cabal run [name] to pick which one.

The REPL

> cabal repl

  • Opens up a REPL (read-eval-print loop) for you to experiment
  • Includes access to all your dependencies and your code
  • Can defer type errors with -fdefer-type-error

Uh, something went wrong

The dreaded ExitFailure 1 during cabal install

  • Check for missing executables in error message
  • Linker failed due to lack of memory
  • Use sandboxes to isolate dependencies
  • Flags to try
    • --reorder-goals – Searches different branches
    • --max-backjumps=1000 – Searches more branches
    • --allow-newer – Ignores upper bounds in build-depends
  • Stackage
  • Delete ~/.ghc, GHC's default user package location
